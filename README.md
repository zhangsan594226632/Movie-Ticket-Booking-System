## 一、项目概述

### 1\.1 项目背景

线下影院购票存在排队时间长、影片信息展示单一、用户无法在线查看影片详情、评论交流缺失、订单管理混乱等痛点。为满足线上电影票选购需求，开发一套前后端分离的电影票购买系统，面向普通观影用户提供影片浏览、分类筛选、在线购票、影评交流、订单管理功能；后台管理员统一管理影片、分类、放映厅、用户评论、用户订单、论坛资讯，实现电影票务线上化、规范化管理。

### 1\.2 开发技术栈

#### 后端技术（SpringBoot）

1. 核心框架：SpringBoot 2\.7\.x，快速开发 RESTful 业务接口

2. 持久层：MyBatis\-Plus，简化分页、多条件查询、增删改查

3. 数据库：MySQL8\.0，存储用户、电影、分类、放映厅、订单、评论、资讯、购物车数据

4. 身份鉴权：JWT Token 区分普通用户、管理员权限，拦截越权访问

5. 文件上传：本地文件存储，支持电影海报、用户头像上传

6. 工具组件：Hutool 工具类，实现表单校验、日期处理、订单编号生成

7. 业务支撑：购物车逻辑、订单状态流转、余额支付、退款、影评回复业务接口

#### 前端技术（Vue3 \+ Element Plus）

1. 核心框架：Vue3 \+ VueRouter \+ Pinia 状态管理

2. UI 组件库：Element Plus（管理后台）、轻量化页面组件（用户前台）

3. 网络请求：Axios 封装请求 / 响应拦截，自动携带登录 Token、统一异常处理

4. 页面功能：轮播图、多条件筛选表格、分页、商品购物车、订单状态标签、评论输入框、图片上传弹窗

5. 交互模块：影片详情页（简介 / 评论 / 选座切换标签）、订单结算页、个人中心订单分类

### 1\.3 系统角色划分

系统分为两大角色，权限完全隔离：

1. **普通观影用户（前台）**：注册登录、浏览首页影片推荐、按分类 / 名称 / 主演筛选电影、查看影片详情、加入购物车、提交订单、余额支付、影评发布、个人中心管理订单（兑换 / 退款）、浏览电影资讯、用户交流论坛。

2. **系统管理员（后台）**：管理全部用户、电影分类、放映厅、电影信息、用户影评（回复评论）、论坛资讯、用户购票订单，完成影片上下架、评论管控、订单审核。

### 1\.4 整体业务流程

用户注册登录 → 首页浏览推荐影片 / 资讯 → 筛选目标电影查看详情（简介、评分、场次）→ 选择票数加入购物车 → 购物车提交订单结算 → 余额支付生成有效订单 → 观影后发布影片评论；管理员后台维护影片、放映厅，查看并回复用户影评，管理全部购票订单，维护电影资讯与论坛内容。

### 2\.1 登录注册模块

#### 2\.1\.1 系统登录页面

- 表单字段：账号、密码输入框，前端非空校验

- 角色区分：普通用户 / 管理员共用登录入口，后端根据账号角色跳转对应页面

- 鉴权逻辑：登录成功返回 JWT Token，本地持久化存储，所有页面请求自动携带 Token，过期强制返回登录页

#### 2\.1\.2 用户注册页面

注册表单完整字段：用户账号、密码、确认密码、用户姓名、头像上传、性别下拉选择、手机号码、地址、邮箱；提交校验两次密码一致、手机号与邮箱格式合法、账号唯一；底部提供已有账号跳转登录入口。

### 2\.2 前台用户核心功能模块

左侧固定导航栏：首页、电影信息、电影资讯、用户交流、购物车、个人中心

#### 2\.2\.1 系统首页

1. 顶部电影海报轮播图，展示主推影片宣传图

2. 电影信息推荐区域：卡片展示影片海报、名称、放映厅、票价，左右切换浏览更多影片

3. 底部电影资讯推荐区：资讯封面 \+ 标题卡片，点击查看完整资讯内容

#### 2\.2\.2 电影信息列表页面

1. 多维度搜索筛选：电影名称、主演、场次时间区间检索

2. 左侧电影分类侧边栏：全部、文艺片、动作片等多类型标签快速筛选影片

3. 影片卡片展示：海报、电影名称、售价，分页加载；点击卡片进入影片详情页

#### 2\.2\.3 电影详情页面

页面分为三大切换标签：电影简介、评论、选座

1. 基础影片信息展示：票价、电影分类、导演、主演、上映日期、片长、评分、场次、放映厅、总座位、收藏量、点击量

2. 购票操作：票数增减、加入购物车、立即预订按钮

3. 评论标签：输入框发布影评，分页查看全部用户评论

4. 选座标签：放映厅座位选择功能

#### 2\.2\.4 购物车与订单提交页面

1. 购物车选中影片后跳转订单结算页

2. 商品清单表格：影片图片、名称、单价、购买数量、总价自动计算

3. 备注输入框、总价展示、余额支付提交按钮，提交后生成购票订单

#### 2\.2\.5 个人中心 \- 商品订单管理

订单按状态分类标签：全部、未支付、已支付、已完成、已退款、已取消

1. 订单列表字段：序号、订单编号、影片商品、购买数量、单价、总价、收货地址、电话、收货人、备注、下单时间

2. 订单操作按钮：已支付订单支持兑换观影、退款申请

#### 2\.2\.6 辅助前台模块

1. **电影资讯**：浏览平台发布的影视行业资讯、新片预告

2. **用户交流**：论坛板块，用户发布观影心得、交流影片感受

### 2\.3 管理员后台功能模块

左侧侧边导航：首页、用户管理、管理员管理、电影信息管理、论坛管理、商品订单

#### 2\.3\.1 电影信息管理子菜单

1. **电影分类管理**：维护影片分类字典（文艺、动作、喜剧等），新增 / 编辑 / 删除分类标签

2. **管理员管理**：后台运维账号新增、权限分配

## 四、核心业务流程

### 4\.1 用户登录鉴权流程

1. 用户输入账号密码提交登录接口，后端校验账号密码与角色

2. 校验通过生成 JWT Token 返回前端，存储 localStorage

3. 所有前台 / 后台接口请求头携带 Token，后端拦截器校验 Token 有效性

4. Token 过期、无访问权限时自动跳转登录页面

### 4\.2 完整购票业务流程

1. 用户注册并登录系统，首页浏览推荐影片，通过分类 / 名称筛选目标电影

2. 进入影片详情查看简介、评分、场次，选择购票数量加入购物车

3. 进入购物车勾选影片，提交订单结算页面

4. 核对订单总价，使用账户余额完成支付，生成**已支付**状态订单

5. 观影完成后，用户进入影片详情评论区发布影评

6. 个人中心可查看全部订单，已支付订单支持兑换观影、发起退款

7. 管理员后台查看所有订单，回复用户影评，维护影片基础信息

### 4\.3 管理员影片管理流程

1. 管理员登录后台，进入电影信息管理模块

2. 维护电影分类、放映厅基础数据

3. 新增 / 编辑电影信息，上传海报、填写票价、场次、主演等信息完成上架

4. 在评论管理页面查看用户影评，针对评论进行官方回复

5. 管理用户购票订单，处理退款审核、查看订单明细

## 五、项目部署说明

### 5\.1 SpringBoot 后端部署

1. MySQL 执行项目 SQL 脚本，创建数据库与全部业务数据表

2. 修改`application.yml`配置文件：数据库连接地址、账号密码、文件上传路径、JWT 加密密钥、服务端口

3. Maven 打包命令：`mvn clean package -DskipTests` 生成可执行 jar 包

4. Windows/Linux 服务器启动：`java -var movie-ticket.jar --port=8080`

### 5\.2 Vue 前端部署

1. 进入前端项目目录，执行`npm install`安装项目依赖

2. 生产打包：`npm run build`，生成 dist 静态资源文件夹

3. Nginx 配置静态托管 dist 目录，反向代理后端 8080 接口地址

4. 访问 Nginx 配置端口进入电影票系统前台，后台路径单独访问管理端

## 六、系统特色与创新点

1. **前后端分离双端架构**：独立用户前台观影购票页面 \+ 管理员全功能后台，权限隔离清晰

2. **完整线上购票闭环**：影片浏览 \- 购物车 \- 订单支付 \- 影评互动 \- 订单售后（退款 / 兑换）全流程业务

3. **多维度影片筛选**：支持名称、主演、场次时间、影片分类多条件组合检索，快速定位影片

4. **影评互动模块**：用户观影后发布评论，管理员后台统一回复，搭建影视交流社区

5. **订单状态分类管理**：个人中心按支付 / 完成 / 退款标签拆分订单，方便用户快速查找

6. **可视化影片推荐**：首页轮播 \+ 卡片推荐热门影片，提升用户浏览体验

7. **轻量化易扩展**：可扩展在线选座、电子票二维码、会员积分、在线支付对接第三方支付等功能

## 七、项目总结

本基于 SpringBoot\+Vue 的雅妮电影票购买系统贴合线上影院票务业务需求，实现普通用户线上选片、购票、影评交流，同时提供完善的后台管理能力，覆盖影片、放映厅、用户、订单、评论、资讯全维度管控。系统分层架构清晰、业务流程完整，适用于计算机毕业设计，也可作为中小型影院线上售票平台基础版本，具备良好的功能拓展与二次开发空间。
